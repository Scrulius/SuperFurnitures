# SuperBlocksDisplays

A Paper plugin for spawning, animating and managing [block-display.com](https://block-display.com) models on your Minecraft server.

Built for **Paper 1.21.x** and **Java 21+**.

---

## How it works

The plugin connects to the block-display.com API to fetch model data. When you spawn a model, it converts the API response into a series of `block_display` and `item_display` entities placed at your location. Each group of entities is tracked internally with a unique ID and tagged via PersistentDataContainer so the plugin can find them again after a restart.

Models are saved to `plugins/SuperBlocksDisplays/spawned.yml` and automatically re-spawned when the server starts. Old entities are cleaned up before re-spawning to prevent duplication.

All command feedback generated by the internal `/summon` and `/data merge` calls is silenced both in the server console and for in-game operators, using a combination of a custom CommandSender wrapper, gamerule toggling, and a Log4j2 filter.

## Installation

Drop `SuperBlocksDisplays-1.0.0.jar` into your server's `plugins/` folder and restart. No configuration needed.

## Commands

All commands use `/bde` as the base. Aliases: `/sbd`, `/blockdisplay`.

Where a command accepts `[name]`, you can pass a model's display name, `nearest`, or leave it empty to auto-target the closest model within 15 blocks.

### Spawning and managing models

**`/bde spawn <source> <name>`** — Spawns a model at your current position. The source can be either a numeric model ID from block-display.com or the name of a model you previously saved to the library. The name you choose is how you'll refer to the model in all other commands. Names must be 2–32 characters, alphanumeric and underscores only, no spaces, and must be unique among active models.

If the model has animations, they start playing automatically in loop mode.

**`/bde remove [name]`** — Removes a model and kills all its display entities. The plugin also sweeps a 50-block radius around the model's origin to catch any orphaned entities that weren't properly tracked.

**`/bde tp [name]`** — Teleports you to a model's spawn point. Keeps your current look direction.

**`/bde list`** — Shows all active models with their name, model ID, coordinates, part count, and animation status. Each entry is clickable to view details.

**`/bde info [name]`** — Detailed view of a single model: location, world, part count, rotation, animation state, speed, and Minecraft version.

**`/bde purge <1-10>`** — Emergency cleanup. Kills every `block_display`, `item_display` and `text_display` entity within the given radius from your position, regardless of whether it belongs to a managed model or not. Radius is capped at 10 to prevent accidents.

### Animation

Models from block-display.com can include keyframe animations stored in their datapack data. The plugin plays these by dispatching the keyframe commands every server tick.

**`/bde anim play <loop|once> [name]`** — Starts playback. In `loop` mode the animation repeats indefinitely. In `once` mode it plays through and stops at the last frame.

**`/bde anim stop [name]`** — Stops playback. The model freezes at its current frame.

**`/bde speed <0.25-4.0> [name]`** — Controls playback speed. At `1` the animation runs at its original speed. Values below 1 slow it down, above 1 speed it up. Uses a float accumulator so fractional speeds like `0.5` (half speed) work smoothly.

**`/bde rotate <yaw> [name]`** — Rotates all entities in the model to the given yaw angle (0–360).

### Library

The library lets you save models permanently so you can spawn them later without needing the original block-display.com ID. This is useful because API links can expire, but a downloaded model in your library is yours forever.

Library files are stored as JSON in `plugins/SuperBlocksDisplays/library/` and are completely separate from the API cache.

**`/bde download <model_id> <name>`** — Fetches the model from the API and saves it to the library under the given name. You can then use that name as the source in `/bde spawn`.

**`/bde library`** — Lists all models in your library. Each entry has a clickable button to quickly spawn it.

**`/bde undownload <name>`** — Permanently deletes a model from the library.

**`/bde clearcache`** — Clears the temporary API response cache in `plugins/SuperBlocksDisplays/cache/`. This does not touch the library.

## Permissions

| Permission | Description | Default |
|---|---|---|
| `superblocksdisp.use` | Grants access to all `/bde` commands | OP |

## Technical details

- Models are spawned using `Bukkit.dispatchCommand` with a custom `SilentCommandSender` that discards all output, preventing console spam.
- The `SEND_COMMAND_FEEDBACK` and `LOG_ADMIN_COMMANDS` gamerules are temporarily disabled during command execution to prevent in-game spam to operators.
- A Log4j2 filter is registered on the root logger to catch any remaining messages like "Modified entity data", "Summoned new", and "No entity was found".
- Entity registration after spawning is not instant. The plugin waits 1 tick to tag entities via PersistentDataContainer, and 3 ticks before marking a model as ready for animation.
- On shutdown, all managed entities are removed from the world. On startup, any leftover entities from a previous crash are cleaned up before re-spawning.
